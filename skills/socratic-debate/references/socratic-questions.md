# 소크라테스 질문 생성 가이드

사회자가 교차 반박 시 추가하는 소크라테스식 질문의 유형과 생성 규칙.

## 질문 유형

| 태그 | 목적 | 패턴 | 예시 |
|------|------|------|------|
| `[전제 검증]` | 숨겨진 가정 노출 | "X가 참이라고 전제하는데, Y인 경우는?" | "트래픽이 선형 증가한다고 가정하는데, 스파이크가 발생하면?" |
| `[역전]` | 반대 상황 탐색 | "반대로 X가 아니라 Y라면 결론은?" | "이 전제가 틀리면 결론은 어떻게 바뀌는가?" |
| `[극단]` | 경계 조건 탐색 | "X를 극단까지 밀면 어떻게 되는가?" | "사용자가 10만 명이 아니라 1억 명이면?" |
| `[대안]` | 놓친 선택지 탐색 | "X 대신 Y를 택했다면?" | "모노리스 대신 이벤트 기반으로 갔다면 이 문제가 존재하는가?" |
| `[귀류]` | 주장의 논리적 귀결 검증 | "X가 맞다면, 그러면 Z도 성립해야 하는데?" | "성능이 중요하다면, 왜 캐싱은 고려하지 않았는가?" |
| `[증거]` | 근거 요구 | "X라고 주장하는데 근거는?" | "복잡도가 낮다고 하는데, 어떤 메트릭으로 측정했는가?" |

## 생성 규칙

1. **쟁점의 핵심 분기점에서 질문한다.** 양쪽이 동의하는 부분이 아니라 갈라지는 지점을 공략한다.
2. **한 번에 1개만.** 여러 질문을 던지면 에이전트가 가장 쉬운 것만 답한다.
3. **구체적이어야 한다.** "이게 맞나?"가 아니라 "X 조건에서 Y가 발생하면 이 주장이 유지되는가?"
4. **답이 열려 있어야 한다.** 예/아니오로 끝나는 질문은 피한다.
5. **상대의 가장 강한 논점을 겨냥한다.** 약한 논점을 공격하면 짚맨 오류에 빠진다.

## 쟁점별 질문 선택 가이드

| 충돌 유형 | 추천 질문 태그 |
|----------|--------------|
| 전제 충돌 (서로 다른 가정) | `[전제 검증]`, `[증거]` |
| 우선순위 차이 (둘 다 맞지만 뭐가 먼저?) | `[극단]`, `[귀류]` |
| 해법 상이 (같은 문제, 다른 답) | `[대안]`, `[역전]` |
