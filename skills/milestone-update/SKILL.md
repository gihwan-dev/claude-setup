# 워크플로우: 마일스톤 진행 상태 업데이트

**목표**: 이번 세션에서 완료한 작업을 `milestone.md`에 반영합니다. 체크박스를 업데이트하고 세션 노트를 추가합니다.

## 1단계: milestone.md 읽기

`milestone.md` 파일을 읽어 현재 마일스톤 상태를 파악합니다.

- 파일이 없으면 사용자에게 알리고 `/milestone` 스킬로 먼저 생성하도록 안내합니다.

## 2단계: 세션 작업 내용 분석

대화 컨텍스트를 기반으로 이번 세션에서 수행한 작업을 정리합니다.

- git diff, git log 등을 활용하여 실제 변경된 코드를 확인합니다.
- 코드 작성 완료 여부와 동작 검증 여부를 판단합니다.

## 3단계: 체크박스 업데이트

milestone.md의 항목들을 검토하고 완료 여부를 판정합니다.

### 판정 기준

| 상태 | 조건 | 표기 |
|------|------|------|
| 완료 | 코드 작성 완료 + 동작 검증됨 | `[x]` |
| 미완료 | 시작 안 함 또는 부분 완료 | `[ ]` |

- **불확실한 항목**은 사용자에게 확인을 요청합니다. 임의로 완료 처리하지 않습니다.
- 이미 `[x]`인 항목은 변경하지 않습니다.

## 4단계: 세션 노트 추가

milestone.md 하단의 참고 노트 섹션에 세션 요약을 추가합니다. 참고 노트 섹션이 없으면 새로 생성합니다.

### 노트 형식

```markdown
---

## 참고 노트

**[YYYY-MM-DD] 세션 요약**:
- 발견된 이슈: ...
- 아키텍처 결정: ...
- 다음 페이즈 영향: ...
```

- 날짜는 오늘 날짜를 사용합니다.
- 해당 사항이 없는 항목은 "없음"으로 기재합니다.
- 기존 노트가 있으면 그 아래에 새 노트를 추가합니다.

## 5단계: 결과 요약

업데이트 결과를 사용자에게 보고합니다.

- 변경된 체크박스 목록 (어떤 항목이 완료 처리되었는지)
- 추가된 세션 노트 내용
- 남은 미완료 항목 수
