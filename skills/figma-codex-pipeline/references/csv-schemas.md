# CSV Schemas

CSV schema definitions used by Figma Codex Pipeline. They act as data contracts between phases.

## 1. components.csv (Phase 1 -> Phase 2)

Generated in Phase 1 (Decompose) and used as the input for `spawn_agents_on_csv` in Phase 2 (Execute).

| Column | Required | Type | Description |
|------|------|------|------|
| `row_id` | Y | string | Unique ID in `comp-001` format with 3-digit zero-padding |
| `page_id` | Y | string | Figma page nodeId, for example `1:2` |
| `component_name` | Y | string | PascalCase component name |
| `figma_selection_url` | Y | string | Figma selection URL for that node |
| `parent_id` | N | string | Parent `row_id`, used for nested structure |
| `depends_on` | N | string | Comma-separated `row_id` list used for Phase 3 integration order. Workers ignore this |
| `target_path` | Y | string | Output file path. **Must be unique per row** |
| `prefetched_data_dir` | Y | string | `.figma-cache/{nodeId}/` path containing `screenshot.png` and `context.json` |
| `design_system_hint` | N | string | Design-system component hints to use, comma-separated |
| `shared_context_ref` | Y | string | Path to `shared_context.json` |
| `acceptance_criteria` | Y | string | One or two sentences describing the acceptance criteria |
| `estimated_complexity` | N | string | `simple` / `moderate` / `complex` |

### Constraints

- `target_path` must be unique inside the CSV to prevent file collisions between workers
- `row_id` must be unique inside the CSV
- `prefetched_data_dir` must contain both `screenshot.png` and `context.json`
- `depends_on` is used only in Phase 3 integration, not by workers

### Example

```csv
row_id,page_id,component_name,figma_selection_url,parent_id,depends_on,target_path,prefetched_data_dir,design_system_hint,shared_context_ref,acceptance_criteria,estimated_complexity
comp-001,1:2,HeroSection,https://figma.com/design/abc?node-id=10-1,,,"src/features/dashboard/ui/HeroSection.tsx",.figma-cache/10:1/,,shared_context.json,"Hero banner and CTA button render correctly",moderate
comp-002,1:2,SidebarNav,https://figma.com/design/abc?node-id=10-2,,,src/features/dashboard/ui/SidebarNav.tsx,.figma-cache/10:2/,"NavItem,Icon",shared_context.json,"Navigation menu items render in the correct order",simple
comp-003,1:2,CardGrid,https://figma.com/design/abc?node-id=10-3,,comp-001,src/features/dashboard/ui/CardGrid.tsx,.figma-cache/10:3/,"Card",shared_context.json,"Card grid layout and repeated items render correctly",complex
```

## 2. Output CSV (Phase 2 Result)

Automatically generated by `spawn_agents_on_csv` from the `output_schema`. The main agent consumes it in Phase 3.

| Column | Type | Description |
|------|------|------|
| `row_id` | string | 1:1 match with `components.csv` `row_id` |
| `status` | string | `success` / `failed` / `skipped` |
| `component_path` | string | Actual generated file path |
| `exports` | string | Exported symbol list, comma-separated |
| `tokens_used` | string | Design tokens used, comma-separated |
| `dependencies` | string | Imported external packages or components, comma-separated |
| `warnings` | string | Warnings encountered during worker execution |
| `error_message` | string | Error message when `status=failed` |

## 3. verification.csv (Phase 3 -> Phase 4)

Generated by the main agent after Phase 3 (Integrate) completes, and used as the input for `spawn_agents_on_csv` in Phase 4 (Verify).

| Column | Required | Type | Description |
|------|------|------|------|
| `row_id` | Y | string | Unique ID in `ver-001` format |
| `check_type` | Y | string | `lint` / `typecheck` / `screenshot` / `viewport` / `accessibility` |
| `target_path` | Y | string | File path to verify |
| `figma_selection_url` | N | string | Figma URL for screenshot comparison, used only by `screenshot` |
| `viewport_width` | N | number | Viewport width for `viewport` checks |
| `component_row_id` | N | string | Reference to `components.csv` `row_id` |
| `shared_context_ref` | Y | string | Path to `shared_context.json` |
| `acceptance_criteria` | Y | string | Acceptance criteria for the check |

### Required or Optional Columns by `check_type`

| `check_type` | Additional Required Fields | Notes |
|------------|-----------|------|
| `lint` | - | ESLint and Prettier validation |
| `typecheck` | - | TypeScript type validation |
| `screenshot` | `figma_selection_url` | Visual comparison between Figma and implementation. Requires Storybook |
| `viewport` | `viewport_width` | Responsive layout validation |
| `accessibility` | - | Basic accessibility checks such as heading order and alt text |

### Storybook-Unavailable Fallback

`screenshot` and `viewport` checks require a Storybook build. When Storybook is not available in the project:
- Replace `screenshot` with `lint` + `typecheck`
- Skip `viewport`

### Example

```csv
row_id,check_type,target_path,figma_selection_url,viewport_width,component_row_id,shared_context_ref,acceptance_criteria
ver-001,lint,src/features/dashboard/ui/HeroSection.tsx,,,,shared_context.json,"No ESLint or Prettier errors"
ver-002,typecheck,src/features/dashboard/ui/HeroSection.tsx,,,,shared_context.json,"No TypeScript compile errors"
ver-003,screenshot,src/features/dashboard/ui/HeroSection.tsx,https://figma.com/design/abc?node-id=10-1,,comp-001,shared_context.json,"Visually matches the Figma design"
ver-004,viewport,src/features/dashboard/ui/CardGrid.tsx,,768,comp-003,shared_context.json,"Cards reflow into two columns at 768px"
ver-005,accessibility,src/features/dashboard/ui/SidebarNav.tsx,,,comp-002,shared_context.json,"nav landmark exists and focus order is correct"
```

## Cross-Reference

- `components.csv` `row_id` is referenced by both the output CSV and `verification.csv` `component_row_id`
- Every CSV points `shared_context_ref` at the same `shared_context.json` file
- The file structure inside `prefetched_data_dir` must match the `prefetch` section in `shared-context-spec.md`
